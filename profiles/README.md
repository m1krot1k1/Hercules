# Optional Profiles

Профили в `profiles/` содержат **project-specific** расширения, которые не должны входить в универсальное ядро `agents/` + `rules/`.

Назначение:
- держать core-конфигурацию переносимой между проектами;
- изолировать узкоспециализированные агенты/правила;
- включать профиль только по явному решению хозяина репозитория.

Типовой путь:

```text
profiles/<profile-name>/
  README.md
  agents/
  rules/
  skills/   # optional
```

Чтобы активировать профиль в конкретном проекте, его артефакты нужно явно скопировать или смёржить в runtime `.cursor/agents/` и `.cursor/rules/`.
